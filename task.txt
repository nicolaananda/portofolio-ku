# Backend Tasks for Blog Feature

The frontend has been updated to support a dynamic Blog/Journal section. We need the following API endpoints and database schema to support this feature.

## 1. Database Schema (Collection: `blogs`)

```json
{
  "title": "String (Required)",
  "slug": "String (Unique, Indexed)",
  "excerpt": "String (Short description for preview)",
  "content": "String (HTML or Markdown content)",
  "coverImage": "String (URL to image)",
  "category": "String (e.g., 'Data Analyst', 'Web Development')",
  "featured": "Boolean (default: false)",
  "readTime": "String (e.g., '5 min read')",
  "author": {
    "name": "String",
    "avatar": "String (URL)",
    "bio": "String"
  },
  "createdAt": "Date",
  "updatedAt": "Date"
}
```

## 2. API Endpoints

### Public Endpoints
- `GET /api/blog`
  - Returns a list of all blog posts.
  - Should support query parameters for filtering (optional but recommended): `?category=...`, `?search=...`
  - Response: `{ status: "success", data: [ ...posts ] }`

- `GET /api/blog/:id` (or `/api/blog/:slug`)
  - Returns a single blog post by ID or Slug.
  - Response: `{ status: "success", data: { ...post } }`

### Admin Endpoints (Protected)
- `POST /api/blog`
  - Create a new blog post.
  - Body: `{ title, content, category, coverImage, ... }`
  - Response: `{ status: "success", data: { ...createdPost } }`

- `PUT /api/blog/:id`
  - Update an existing blog post.
  - Body: `{ title, content, ... }` (Partial updates)
  - Response: `{ status: "success", data: { ...updatedPost } }`

- `DELETE /api/blog/:id`
  - Delete a blog post.
  - Response: `{ status: "success", message: "Post deleted" }`

## 3. Notes
- Ensure CORS is configured to allow requests from the frontend domain.
- Image upload handling should be consistent with the Portfolio image upload (returning a URL).
